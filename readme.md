## 🎯 Описание проекта

### Цель создания

Приложение создано как тренировочный полигон для тестирования. На этом проекте можно практиковать:

- Функциональное тестирование (позитивные и негативные сценарии).
- UI/UX тестирование.
- API тестирование (Postman, REST API).
- Тестирование базы данных (SQL-запросы, валидация данных).
- Нагрузочное тестирование.
- Тестирование безопасности (JWT, SQL-инъекции, XSS).
- Кроссбраузерное тестирование.
- Адаптивное тестирование.

### Основной функционал

- Регистрация новых пользователей.
- Аутентификация существующих пользователей.
- Создание заметок (todo).
- Отметка заметок как выполненных и невыполненных.
- Удаление заметок.
- Просмотр всех заметок текущего пользователя.

## 🔌 API эндпоинты

### Базовый URL

- Локально: `http://localhost:3000`
- Продакшен: `https://your-app.onrender.com`

### Аутентификация

#### `POST /auth/register` — Регистрация

**Заголовки:**
```json
{
  "Content-Type": "application/json"
}
```

**Тело запроса:**
```json
{
  "username": "string (required, min 1 char)",
  "password": "string (required, min 4 chars)"
}
```

**Успешный ответ (200):**
```json
{
  "success": true,
  "token": "jwt_token_string",
  "redirect": "/html/main.html",
  "user": {
    "id": 1,
    "username": "testuser"
  }
}
```

**Ошибки:**

| Код | Условие | Ответ |
|---|---|---|
| 400 | Пустые поля | `{"error": "Заполните все поля"}` |
| 400 | Пароль < 4 символов | `{"error": "Пароль должен быть минимум 4 символа"}` |
| 400 | Пользователь существует | `{"error": "Пользователь уже существует"}` |
| 500 | Ошибка сервера | `{"error": "Внутренняя ошибка сервера"}` |

#### `POST /auth/login` — Вход

**Тело запроса:**
```json
{
  "username": "string",
  "password": "string"
}
```

**Успешный ответ (200):** аналогичен регистрации.

**Ошибки:**

| Код | Ответ |
|---|---|
| 401 | `{"error": "Неверное имя пользователя или пароль"}` |

### Заметки (требуют Bearer токен)

**Заголовок авторизации:**
```http
Authorization: Bearer <token>
```

#### `GET /api/todos` — Получить все заметки

**Успешный ответ (200):**
```json
[
  {
    "id": 1,
    "text": "Купить хлеб",
    "completed": false,
    "created_at": "2026-01-15T10:30:00Z"
  }
]
```

**Ошибки:**

| Код | Ответ |
|---|---|
| 401 | `{"error": "Требуется авторизация"}` |
| 403 | `{"error": "Недействительный токен"}` |

#### `POST /api/todos` — Создать заметку

**Тело запроса:**
```json
{
  "text": "string (required, min 1 char, max 500 chars)"
}
```

**Успешный ответ (200):**
```json
{
  "id": 1,
  "text": "Купить хлеб",
  "completed": false,
  "created_at": "2026-01-15T10:30:00Z"
}
```

**Ошибка (400):**
```json
{"error": "Текст заметки не может быть пустым"}
```

#### `PUT /api/todos/:id` — Обновить статус

**Тело запроса:**
```json
{
  "completed": true
}
```

**Успешный ответ (200):**
```json
{"success": true}
```

**Ошибка (404):**
```json
{"error": "Заметка не найдена"}
```

#### `DELETE /api/todos/:id` — Удалить заметку

**Успешный ответ (200):**
```json
{"success": true}
```

**Ошибка (404):**
```json
{"error": "Заметка не найдена"}
```

## 🗄️ Структура базы данных

### Таблица `users`

| Колонка | Тип | Ограничения | Описание |
|---|---|---|---|
| id | SERIAL | PRIMARY KEY | Уникальный ID пользователя |
| username | VARCHAR(50) | UNIQUE, NOT NULL | Имя пользователя |
| password_hash | VARCHAR(255) | NOT NULL | Хеш пароля (bcrypt) |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | Дата регистрации |

### Таблица `todos`

| Колонка | Тип | Ограничения | Описание |
|---|---|---|---|
| id | SERIAL | PRIMARY KEY | Уникальный ID заметки |
| user_id | INTEGER | REFERENCES users(id) ON DELETE CASCADE | Владелец заметки |
| text | VARCHAR(500) | NOT NULL | Текст заметки |
| completed | BOOLEAN | DEFAULT FALSE | Статус выполнения |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | Дата создания |

### Индексы

```sql
CREATE INDEX idx_todos_user_id ON todos(user_id);
CREATE INDEX idx_users_username ON users(username);
```

## 🚀 Установка и запуск

### Требования

- Node.js 18+
- PostgreSQL 14+
- npm или yarn

### Локальный запуск

```bash
# 1. Клонировать репозиторий
git clone https://github.com/ваш-username/todo-list-app.git
cd todo-list-app

# 2. Установить зависимости бэкенда
cd backend
npm install

# 3. Создать базу данных PostgreSQL
# Открыть psql и выполнить:
CREATE DATABASE todo_app;

# 4. Создать таблицы
# См. раздел "Структура базы данных"

# 5. Создать файл .env в папке backend
cat > .env << EOF
PORT=3000
DB_USER=postgres
DB_PASSWORD=ваш_пароль
DB_HOST=localhost
DB_PORT=5432
DB_NAME=todo_app
JWT_SECRET=сгенерировать_секретный_ключ
EOF

# 6. Запустить сервер
node server.js

# или в режиме разработки:
npx nodemon server.js

# 7. Открыть в браузере
# http://localhost:3000/html/index.html
```
